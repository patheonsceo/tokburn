# v2.1 Ideas

## Project-aware quips

The status line stdin JSON includes `workspace.current_dir`. The Tokemon could
occasionally reference the project name in its quips, making it feel like it
actually knows what you're working on.

Examples:
- Sassy: `"still grinding on tokburn huh"`, `"back to api-server. bold."` 
- Hype: `"TOKBURN SESSION!! LETS BUILD!!"`
- Anxious: `"oh we're in tokburn again... last time was intense..."`

Implementation:
- Extract `path.basename(workspace.current_dir)` in statusline.js
- Add a `project` pool to each personality (5-8 templates with `{project}` placeholder)
- 10-20% chance per minute of showing a project quip instead of mood quip
- Template: `getMessage()` checks if project quip should fire, replaces `{project}` with dir name

This makes the Tokemon feel aware of context without being creepy — it only knows
the directory name, nothing about the code itself.
